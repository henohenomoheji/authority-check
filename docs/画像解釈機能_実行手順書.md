# 画像解釈機能 実行手順書

## 1. 目的
authority-check の画像解釈機能をローカル環境で起動し、画像をアップロードして説明文を生成するまでの手順をまとめる。

## 2. 前提条件
- Python 3.9 以上
- `pip` が利用可能であること
- Azure OpenAI を利用できること
- Azure OpenAI のエンドポイント、API キー、デプロイ名を取得済みであること

## 3. 初回セットアップ

### 3.1 リポジトリへ移動
```bash
cd authority-check
```

### 3.2 仮想環境の作成と有効化
```bash
python -m venv .venv
source .venv/bin/activate
```

### 3.3 依存関係のインストール
```bash
pip install -r requirements.txt
```

## 4. 環境変数設定

### 4.1 `.env` ファイルを準備
リポジトリ直下の `.env.example` をコピーして `.env` を作成する。

```bash
cp .env.example .env
```

### 4.2 `.env` に設定する項目
```dotenv
AZURE_OPENAI_API_KEY=your_azure_openai_key
AZURE_OPENAI_ENDPOINT=https://your-resource-name.openai.azure.com
AZURE_OPENAI_DEPLOYMENT_NAME=your-vision-deployment
SSL_CERT_FILE=/absolute/path/to/certifi/cacert.pem
VISION_TIMEOUT_SEC=10
MAX_IMAGE_SIZE_MB=10
```

`SSL_CERT_FILE` は、Python 実行時に証明書検証エラーが発生する場合に設定する。`certifi` をインストール後、以下のコマンドでパスを確認できる。

```bash
python -c "import certifi; print(certifi.where())"
```

### 4.3 各設定値の説明

| 変数名 | 必須 | 説明 |
|---|---|---|
| `AZURE_OPENAI_API_KEY` | Yes | Azure OpenAI の API キー |
| `AZURE_OPENAI_ENDPOINT` | Yes | Azure OpenAI リソースのエンドポイント |
| `AZURE_OPENAI_DEPLOYMENT_NAME` | Yes | 画像解釈に利用する Azure OpenAI のデプロイ名 |
| `SSL_CERT_FILE` | No | Python が参照する CA 証明書ファイルのパス |
| `VISION_TIMEOUT_SEC` | No | API 呼び出しのタイムアウト秒数 |
| `MAX_IMAGE_SIZE_MB` | No | アップロード可能な画像サイズ上限 |

## 5. 起動手順

### 5.1 Streamlit アプリを起動
```bash
streamlit run app.py
```

### 5.2 ブラウザでアクセス
通常は以下の URL が表示される。

```text
http://localhost:8501
```

### 5.3 画像解釈ページを開く
左メニューから `画像解釈` ページを選択する。

## 6. 実行手順
1. `画像ファイルを選択` から PNG / JPG / JPEG / WebP のいずれかをアップロードする。
2. 必要に応じて `指示プロンプト` を入力する。
3. `解釈実行` ボタンを押下する。
4. 生成された説明文と実行メタ情報を確認する。

## 7. 動作確認の観点
- 画像未選択時にエラーが表示されること
- 10MB 超の画像でサイズエラーになること
- `.env` 未設定時に設定エラーが表示されること
- 正常時に説明文、`request_id`、`model_name`、`latency_ms` が表示されること

## 8. トラブルシュート

### API キー未設定
- 症状: `設定エラー: AZURE_OPENAI_API_KEY が設定されていません。`
- 対応: `.env` の `AZURE_OPENAI_API_KEY` を設定して再起動する

### エンドポイント未設定
- 症状: `設定エラー: AZURE_OPENAI_ENDPOINT が設定されていません。`
- 対応: `.env` の `AZURE_OPENAI_ENDPOINT` を設定して再起動する

### デプロイ名未設定
- 症状: `設定エラー: AZURE_OPENAI_DEPLOYMENT_NAME が設定されていません。`
- 対応: `.env` の `AZURE_OPENAI_DEPLOYMENT_NAME` を設定して再起動する

### タイムアウト
- 症状: `時間をおいて再実行してください。`
- 対応: ネットワーク状態を確認し、必要に応じて `VISION_TIMEOUT_SEC` を見直す

### SSL 証明書エラー
- 症状: `接続エラー: Azure OpenAI への SSL 接続に失敗しました。`
- 対応: `certifi` をインストールし、`.env` の `SSL_CERT_FILE` に `python -c "import certifi; print(certifi.where())"` で確認したパスを設定して再起動する

### 対応外の画像形式
- 症状: `対応外の画像形式です。`
- 対応: PNG / JPG / JPEG / WebP のいずれかに変換して再実行する
