# 画像解釈機能 要件定義書（素案 v0.2）

## 1. 文書情報
- 文書名: 画像解釈機能 要件定義書
- 版数: v0.2（素案改訂）
- 対象システム: authority-check（Streamlit）
- 作成目的: 実装前に、MVPで満たすべき要件を合意可能な粒度で定義する

## 2. 目的
ユーザーが入力した **画像ファイル** と **指示プロンプト** をもとに、画像内容を自然言語で説明する文章を生成する。

## 3. 背景・課題
- 現状は権限制御確認が主用途で、画像解釈機能は未提供。
- 手作業での説明文作成に時間がかかるため、業務効率化ニーズがある。
- まずはMVPとして「1画像 + 1プロンプト + 1説明文」を対象に提供する。

## 4. スコープ

### 4.1 In Scope（MVP）
- 画像1枚（PNG/JPEG/WebP）をアップロード
- 指示プロンプトを入力（任意）
- 画像 + プロンプトで説明文を生成
- 結果表示、エラーメッセージ表示
- 実行ログ（request_id / 成否 / 所要時間）を記録

### 4.2 Out of Scope（MVPでは非対応）
- 複数画像一括処理
- 画像編集/加工
- OCR抽出の正確性保証
- 外部システム連携（Slack、メール通知）
- 学習/再学習（Fine-tuning）

## 5. 想定ユーザー
- 社内業務担当者
- 画像に含まれる内容を短時間で文章化したい利用者

## 6. ユースケース（主要シナリオ）
1. ユーザーが画像をアップロードする。
2. ユーザーが指示プロンプトを入力する（例: 「200文字以内で説明」）。
3. ユーザーが「解釈実行」を押下する。
4. システムが説明文を生成して画面表示する。
5. ユーザーは必要に応じてプロンプトを修正し再実行する。

## 7. 機能要件

| ID | 要件 | 詳細 | 優先度 |
|---|---|---|---|
| FR-01 | 画像入力 | 対応形式: png/jpg/jpeg/webp、最大10MB、1ファイルのみ | Must |
| FR-02 | プロンプト入力 | 任意入力。未入力時はデフォルト文を使用 | Must |
| FR-03 | 解釈実行 | 画像 + プロンプトをモデルに送信し説明文を生成 | Must |
| FR-04 | 結果表示 | 生成文を画面表示。処理中はローディング表示 | Must |
| FR-05 | エラー表示 | ファイル不正/サイズ超過/API失敗をユーザー向け文言で表示 | Must |
| FR-06 | ログ記録 | request_id、成否、処理時間、エラー種別を記録 | Must |
| FR-07 | 再実行 | 同一画像でプロンプト変更後に再実行可能 | Should |

## 8. 非機能要件

| ID | 区分 | 要件 |
|---|---|---|
| NFR-01 | 性能 | 95パーセンタイルで10秒以内に応答（外部API正常時） |
| NFR-02 | 可用性 | 外部APIの一時障害時に失敗を明示し、利用者が再試行可能 |
| NFR-03 | セキュリティ | APIキーは環境変数管理、コード直書き禁止 |
| NFR-04 | 保守性 | UI層とサービス層を分離し、モデル差し替え容易な構成 |
| NFR-05 | 監査性 | request_idで実行追跡できるログを残す |

## 9. 入出力仕様（MVP）

### 9.1 入力
- image_file: バイナリ画像（必須）
- instruction_prompt: 文字列（任意、最大2,000文字）

### 9.2 出力
- description_text: 生成説明文（必須）
- request_id: 実行識別子（必須）
- model_name: 利用モデル名（必須）
- latency_ms: 実行時間（必須）

## 10. エラー要件

| エラーケース | ユーザー表示 | ログ出力 |
|---|---|---|
| 未対応拡張子 | 「対応外の画像形式です」 | error_type=invalid_format |
| サイズ超過 | 「画像サイズが上限を超えています」 | error_type=image_too_large |
| APIタイムアウト | 「時間をおいて再実行してください」 | error_type=model_timeout |
| API障害 | 「現在利用できません」 | error_type=model_unavailable |

## 11. 受け入れ基準（Acceptance Criteria）
- AC-01: 有効画像 + 任意プロンプトで説明文が表示される。
- AC-02: プロンプト空欄時でもデフォルト文で処理できる。
- AC-03: 形式不正/サイズ超過時に実行前にエラー表示される。
- AC-04: APIキー未設定時に実行不能であることを検知・通知できる。
- AC-05: 成功/失敗ともに request_id を含むログが残る。

## 12. 制約・前提
- 外部のVision対応モデルAPIが利用可能であること。
- ネットワークおよびAPI利用制限（レート制限・課金）を考慮すること。
- 生成内容の正確性は確率的であり、業務最終判断はユーザー責任とすること。

## 13. リスクと対応方針
- モデルの誤認識 → 画面上に「自動生成である」旨を明記する。
- コスト増大 → 画像サイズ上限・タイムアウト・将来の利用回数制限で制御。
- レスポンス遅延 → タイムアウト設定・再試行導線を提供。
